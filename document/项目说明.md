# 绿果付费短剧 · 项目说明

## 一、项目定位

**绿果付费短剧** 是一套完整的短剧付费平台解决方案，覆盖：

- **用户侧**：微信小程序看剧、发弹幕、购买短剧
- **运营侧**：Web 后台管理内容与营销活动
- **服务端**：Spring Cloud 微服务支撑高并发与可扩展架构

适用于短剧平台原型验证、全栈项目展示与技术学习。

---

## 二、业务场景

### 典型用户路径

```
打开小程序 → 浏览首页推荐 → 进入短剧详情 → 播放试看/正片
    → 发送弹幕 → 加入购物车 → 下单付费 → 我的订单查看
```

### 典型运营路径

```
登录后台 → 上架短剧（分类/季/集）→ 上传封面与视频到 MinIO
    → 配置首页轮播与秒杀活动 → 管理用户订单与评论举报
```

## 四、中间件职责

| 中间件 | 用途 |
|--------|------|
| Nacos | 服务注册发现、集中配置（数据库、Redis、Token 白名单等） |
| MySQL | 业务持久化，四库分离 |
| Redis | Token 会话、秒杀缓存、热点短剧信息 |
| Elasticsearch | 弹幕存储与检索、短剧搜索 |
| MinIO | 封面、剧集视频、头像、轮播图等静态资源 |
| RocketMQ | 异步下单、订单消息等 |
| WebSocket | 弹幕实时广播 |

## 五、接口访问规范

所有 HTTP 接口经网关 `ml-gateway` 转发，路径格式：

```
http://{gateway}/{service-name}/api/v1/{module}/{action}
```

示例：

```
GET  /course-server/api/v1/course/page          # 短剧分页
GET  /course-server/api/v1/episode/searchBarrage/course/1  # 课程弹幕
POST /user-server/api/v1/user/loginByAccount    # 登录
WS   ws://{host}:24106/api/v1/barrage/{userId}  # 弹幕
```

请求头需携带 `token`（登录后获取，白名单接口除外）。
